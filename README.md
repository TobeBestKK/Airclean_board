# Airclean_board

基于 `CMS79F723` 的空气净化器主控固件工程，负责触摸按键、风机控制、TM1628A 显示、TP-401W 异味/空气质量传感器、EEPROM 数据保存、涂鸦 Wi-Fi 协议和 SU-03T 语音指令处理。

本 README 按当前工作区源码整理。硬件引脚以 `board.h` 为准，协议与数据点以 `tuya_protocol.h` / `tuya_protocol.c` 为准；仓库没有包含完整原理图、面板丝印映射、烧录器配置或固定版本的 IDE 安装包。

## 项目定位

- 固件入口：`main.c`
- 工程文件：`Airclean.cmwproj`
- 目标器件：`CMS79F723`
- 编译器配置：`CMSC`
- 主频配置：内部 `16MHz`
- 触摸库：`CMS_CheckTouch_PT_79F72x_V1.02.lpp`
- 主要开发语言：C，以及 CMS 编译器支持的少量汇编指令

这是一个面向实际单片机板卡的固件工程，不是可以直接在电脑上运行的应用程序。构建需要安装与 `CMS79F723` 和 `CMSC` 工程格式兼容的 CMS 开发环境。

## 功能概览

### 本地控制

- 电源开关：触摸按键或语音控制开关机。
- 风机三档：`OFF`、低档、中档、高档，档位值为 `0~3`。
- 定时关机：定时范围为 `0~24` 小时；短按按小时递增，长按清除定时。
- 滤网指示：支持滤网状态指示、使用率显示和滤网使用率清零。
- 显示模式：支持 PM2.5 数值与滤网使用率之间切换。
- 屏幕亮度：支持低、中、高亮度以及省电熄屏档位。
- 开机后传感器预热：TP-401W 传感器预热期间显示剩余秒数。

### 风机与定时器

- `RB0` 控制风机供电使能。
- `RB3` 输出软件 PWM。
- Timer2 同时为触摸扫描和风机 PWM 提供周期中断。
- PWM 周期为 `10` 个 Timer2 tick：低档占空比 `6/10`，中档 `8/10`，高档 `10/10`。
- Timer0 以轮询方式累计约 1 秒事件，用于定时倒计时、PM2.5 刷新、滤网使用率累计和传感器预热倒计时。

### 传感器与滤网数据

- TP-401W 传感器数据输入：`RB2/AN13`。
- 传感器供电控制：`RB1`，低电平供电。
- ADC 采用 6 次采样，去除最大值和最小值后取平均值。
- 传感器数据使用一级 IIR 低通滤波，系数为 `GAS_FILTER_ALPHA = 64`。
- 传感器有效读数接口返回 `0~500`，`GAS_PM25_INVALID = 1000` 表示无效、未上电、开路、短路或采样超时。
- 传感器预热时间为 `30` 秒。
- 滤网使用率保存于 CMS79F723 内置 Data EEPROM，范围为 `0~100`。
- EEPROM 未初始化或数据越界时，滤网使用率默认恢复为 `40`。
- 滤网使用率超过 `90` 时，固件通过 `led_force_mask` 强制叠加告警指示。

### TM1628A 显示

`tm1628.c` 提供以下显示能力：

- 风速档位显示。
- 定时小时显示。
- 默认待机/运行显示。
- PM2.5 数值显示。
- 滤网使用率显示。
- LED 指示灯控制。
- 亮度设置、全亮、全灭和 walking test。

## 触摸按键逻辑

当前逻辑按键掩码定义在 `key.h`：

- `K1`：定时。短按增加 1 小时，长按清除定时。
- `K2`：滤网/指示功能。短按切换 PM2.5 与滤网使用率显示，长按切换滤网指示；在相应状态下可清零滤网使用率。
- `K3`：电源。短按切换开关机，长按约 2 秒进入 AP 配网模式。
- `K4`：风速。短按循环切换 `OFF → 低档 → 中档 → 高档 → OFF`，长按恢复为 `OFF`。

语音控制复用同一套本地状态机，因此语音、触摸、云端 DP 下发会共同更新显示、风机、定时和指示灯状态。

## 当前引脚定义

以下内容来自 `board.h`，表示源码中的板级定义：

- `RA0`：`KEY_SPEED`
- `RA1`：`KEY_FILTER`
- `RA2`：`KEY_POWER`
- `RA3`：`KEY_TIMER`
- `RA4`：`TM1628_DIO`
- `RA5`：`TM1628_CLK`
- `RA6`：`TM1628_STB`
- `RB0`：`FAN_VCC`
- `RB1`：`GAS_VCC`
- `RB2/AN13`：`GAS_DATA`
- `RB3`：`FAN_PWM`
- `RC0`：UART TX
- `RC1`：UART RX

注意：源码同时使用了 `KEY_SPEED`、`KEY_FILTER`、`KEY_POWER`、`KEY_TIMER` 和 `K1~K4` 两套命名。`K1~K4` 是当前控制逻辑中的按键编号，最终物理按键丝印仍应结合实际面板和原理图确认。

## 涂鸦 Wi-Fi 与语音协议

### UART 参数

- MCU 工作模式：`WIFI_MODE_MCU`
- UART：`RC0/TX`、`RC1/RX`
- 波特率：`9600bps`
- 当前配置：`16MHz` 主频下 `SPBRG = 103`
- 涂鸦协议帧头：`0x55 0xAA`
- 协议版本：`0x03`
- UART 接收使用中断环形缓冲区，主循环调用 `WIFI_Process()` 完成协议解析和应答。

### 数据点定义

数据点 ID 定义在 `tuya_protocol.h`：

- `DPID_POWER = 101`：电源，`bool`
- `DPID_TIMER = 102`：定时，`value`，范围 `0~24`
- `DPID_FAN_SPEED = 103`：风速，`enum`，范围 `0~3`
- `DPID_INDICATOR = 104`：状态/滤网指示，`bool`
- `DPID_BRIGHTNESS = 105`：亮度，`enum`，范围 `0~3`
- `DPID_PM25 = 106`：PM2.5，`value`，范围 `0~999`
- `DPID_FILTER_USAGE = 107`：滤网使用率，`value`，范围 `0~100`

固件支持单个 DP 上报和全部 DP 分时上报，云端下发的电源、定时、风速、指示和亮度数据会在主循环中同步到本地状态机。

### SU-03T 语音命令

语音模块与 Wi-Fi 模块复用同一 UART。协议解析器将单字节语音命令从涂鸦协议帧中分流到语音队列：

- `0xA0`：唤醒
- `0xA1`：开机
- `0xA2`：关机
- `0xA3`：风速增加
- `0xA4`：风速降低
- `0xA5`：增加定时
- `0xA6`：滤网/指示功能

生产使用前，应确认 `tuya_protocol.c` 中的 `ProductkeyMcuv[]` 已替换为涂鸦 IoT 平台对应产品的实际产品信息。

## 构建方式

仓库当前没有 Makefile、命令行构建脚本或 CI 配置，推荐使用与工程格式兼容的 CMS IDE 完成构建。

### 前置条件

- 安装支持 `CMS79F723` 的 CMS 开发环境。
- 安装或配置 `CMSC` 编译器和对应器件头文件，使 `#include <cms.h>` 可用。
- 确认触摸库 `CMS_CheckTouch_PT_79F72x_V1.02.lpp` 能被工程加载。
- 准备与 `CMS79F723` 匹配的下载/烧录工具。

### IDE 构建步骤

1. 使用 CMS IDE 打开 `Airclean.cmwproj`。
2. 检查目标器件为 `CMS79F723`，编译器为 `CMSC`。
3. 检查工程中的源文件、头文件和 `.lpp` 触摸库均已加载。
4. 执行 Build/Rebuild。
5. 使用生成的 `HEX` 或 `BIN` 文件，通过匹配的烧录工具下载到目标板。

工程文件中的输出目录当前使用绝对路径：

```text
K:\School\Smart_home_appliances\Airclean_board
K:\School\Smart_home_appliances\Airclean_board\output
```

在其他电脑上使用前，应在 IDE 中修改为本机路径。构建生成的 `output/`、`*.hex`、`*.bin` 等文件已被 `.gitignore` 排除，通常不应提交到源码仓库。

## 上电与验证建议

建议每次修改后至少完成以下板级验证：

1. 上电后确认默认处于关机状态，风机和传感器均未持续输出。
2. 短按 `K3`，确认风机电源、TM1628A 默认显示和 30 秒传感器预热倒计时正常。
3. 预热完成后确认 PM2.5 能刷新；拔除或异常连接传感器时应显示无效状态，而不是沿用旧值。
4. 短按 `K4` 逐档切换风速，确认 `RB3` 软件 PWM 和 `LED4`/档位显示同步。
5. 短按 `K1` 设置定时，确认倒计时结束后自动关机并上报电源、定时状态。
6. 短按 `K2` 切换 PM2.5/滤网使用率显示，长按验证滤网指示和使用率清零逻辑。
7. 长按 `K3`，确认 Wi-Fi 模块进入 AP 配网模式。
8. 验证云端 DP 下发、DP 上报，以及 `0xA0~0xA6` 语音命令与本地按键行为一致。
9. 连续运行一段时间，观察 EEPROM 写入、Timer2 中断、UART 接收和风机 PWM 是否互相影响。

当前仓库没有自动化单元测试，功能验证主要依赖 CMS 编译结果、生成物检查和真实硬件测试。

## 开发注意事项

- 修改引脚时，优先同步检查 `board.h`、`Board_Init()`、触摸配置、显示驱动和相关状态机。
- Timer2 同时服务触摸采样和软件 PWM，调整中断周期时需要同时验证触摸灵敏度和风机占空比。
- `gas_sensor.c` 的 ADC 采样会暂时关闭外设中断，以避免 ADC 寄存器被 Timer2 ISR 并发修改；改动采样流程时需重新验证中断时序。
- 涂鸦 DP 的类型、范围和 ID 必须与云端产品定义保持一致。
- Wi-Fi 和 SU-03T 共用 UART，新增串口协议时需要避免与 `0x55` 帧头和 `0xA0~0xA6` 语音字节冲突。
- 仓库当前未提供完整原理图、PCB 版本和烧录工具型号；这些信息应在产品交付或公开发布前补充确认。
